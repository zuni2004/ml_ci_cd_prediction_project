Project Overview

This project predicts CI/CD build failures before execution using commit metadata from Travis CI and GitHub. By analyzing 62 features spanning code changes, team metrics, and historical patterns, the model enables pre-build gates to save computational resources and development time.

Target Variable: build_successful (Binary: True=Pass, False=Fail)
Data Scope: 62 features covering GitHub metadata, Travis CI logs, code metrics, and repository statistics

Feature Preprocessing Pipeline

A systematic approach to preparing your 62 features for machine learning.

Recommended Tool Stack: pandas, scikit-learn, category_encoders, imblearn
Step 1: Data Loading & Exploration
import pandas as pd import numpy as np from sklearn.preprocessing import StandardScaler, MinMaxScaler from sklearn.compose import ColumnTransformer from sklearn.pipeline import Pipeline # Load data df = pd.read_csv('build_data.csv') # Separate features and target X = df.drop('build_successful', axis=1) y = df['build_successful'] # Check missing values print(X.isnull().sum()) print(f"Class balance: {y.value_counts()}")
Step 2: Drop Identifiers & Non-Predictive Features
# Features to DROP (16 total) drop_features = [ 'tr_build_id', 'tr_job_id', 'git_prev_built_commit', 'git_all_built_commits', 'git_trigger_commit', 'tr_original_commit', 'gh_commits_in_push', 'tr_log_tests_failed', 'gh_project_name' # High cardinality ] X = X.drop(drop_features, axis=1) print(f"Features after dropping: {X.shape[1]}")
Step 3: Handle Missing Values
# Strategy: Fill numerical with median, categorical with mode from sklearn.impute import SimpleImputer numerical_cols = X.select_dtypes(include=['int64', 'float64']).columns categorical_cols = X.select_dtypes(include=['object']).columns # Numerical imputation num_imputer = SimpleImputer(strategy='median') X[numerical_cols] = num_imputer.fit_transform(X[numerical_cols]) # Categorical imputation cat_imputer = SimpleImputer(strategy='most_frequent') X[categorical_cols] = cat_imputer.fit_transform(X[categorical_cols])
Step 4: Extract Temporal Features
# From: gh_pushed_at, gh_build_started_at, gh_pr_created_at, gh_first_commit_created_at def extract_temporal_features(df, date_col): """Extract hour, day_of_week, is_weekend from datetime""" df[f'{date_col}_hour'] = pd.to_datetime(df[date_col]).dt.hour df[f'{date_col}_dow'] = pd.to_datetime(df[date_col]).dt.dayofweek df[f'{date_col}_weekend'] = (df[f'{date_col}_dow'] >= 5).astype(int) return df temporal_cols = ['gh_pushed_at', 'gh_build_started_at'] for col in temporal_cols: if col in X.columns: X = extract_temporal_features(X, col) X = X.drop(col, axis=1) # Drop original
Step 5: Feature Encoding

Nominal (Low Cardinality) → One-Hot Encoding

from sklearn.preprocessing import OneHotEncoder nominal_low_card = ['gh_lang', 'tr_log_lan', 'git_merged_with', 'git_branch', 'tr_virtual_merged_into'] encoder_oh = OneHotEncoder(sparse_output=False, handle_unknown='ignore') X_encoded_oh = encoder_oh.fit_transform(X[nominal_low_card]) X_encoded_oh_df = pd.DataFrame(X_encoded_oh, columns=encoder_oh.get_feature_names_out(nominal_low_card)) X = X.drop(nominal_low_card, axis=1) X = pd.concat([X, X_encoded_oh_df], axis=1)

Binary → Label Encoding (0/1)

binary_features = ['gh_is_pr', 'gh_by_core_team_member', 'tr_log_bool_tests_ran', 'tr_log_bool_tests_failed'] for col in binary_features: if col in X.columns: X[col] = (X[col] == True).astype(int) # Convert TRUE/FALSE to 1/0

Nominal (High Cardinality) → Frequency Encoding

from category_encoders import TargetEncoder # For tr_log_frameworks (multiple frameworks) high_card_features = ['tr_log_frameworks'] target_encoder = TargetEncoder(cols=high_card_features) X[high_card_features] = target_encoder.fit_transform(X[high_card_features], y)
Step 6: Numerical Scaling
# Identify numerical features numerical_features = X.select_dtypes(include=['int64', 'float64']).columns.tolist() # StandardScaler for most features (good for tree-based & linear models) scaler = StandardScaler() X[numerical_features] = scaler.fit_transform(X[numerical_features]) # Alternative: MinMaxScaler for bounded features [0,1] # Useful for neural networks minmax_scaler = MinMaxScaler() X[numerical_features] = minmax_scaler.fit_transform(X[numerical_features])
Step 7: Feature Engineering (Advanced)

Create Interaction Features

# Test coverage ratio X['test_to_src_ratio'] = X['git_diff_test_churn'] / (X['git_diff_src_churn'] + 1) # Code change risk X['total_diff_files'] = (X['gh_diff_files_added'] + X['gh_diff_files_deleted'] + X['gh_diff_files_modified']) # Commit complexity X['commit_entropy'] = -X['gh_num_commits_in_push'] * np.log(X['gh_num_commits_in_push'] + 1)

Handling Class Imbalance

from imblearn.over_sampling import SMOTE smote = SMOTE(random_state=42) X_balanced, y_balanced = smote.fit_resample(X, y) print(f"Class distribution after SMOTE:") print(y_balanced.value_counts())

Evaluation Metrics
Metric Formula When to Use Target
Accuracy (TP+TN)/(TP+TN+FP+FN) Balanced classes only > 0.85
Precision TP/(TP+FP) Minimize false alarms > 0.80
Recall TP/(TP+FN) ← Critical for us Catch all failures > 0.90
F1-Score 2*(Precision*Recall)/(Precision+Recall) Balance precision & recall > 0.85
ROC-AUC Area under curve Overall discrimination ability > 0.88
Common Pitfalls & Solutions
Pitfall: Data leakage (using test-time features like test results for training)
Solution: Use only commit metadata available BEFORE execution
Pitfall: Imbalanced classes (95% pass, 5% fail)
Solution: Use SMOTE, class_weight, or stratified splitting
Pitfall: Not scaling numerical features before tree-based models
Solution: Always scale for consistency, even if not required mathematically
Pitfall: Fitting scaler/encoder on full dataset
Solution: Fit on train set only, transform test set with train parameters
Pitfall: Using high-cardinality identifiers as features
Solution: Always drop identifiers and hash-based features
