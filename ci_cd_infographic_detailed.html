📊 Project Overview

The CI/CD Build Failure Predictor is a machine learning system that predicts whether a continuous integration build will succeed or fail before execution. By analyzing commit metadata, code churn, and team metrics, the model enables teams to identify high-risk builds and take preventive action, reducing wasted computational resources and accelerating development velocity.

⚙️ Data Preprocessing Pipeline

The preprocessing pipeline follows ML best practices with careful attention to preventing data leakage. All encoders and scalers are fitted on the training set only and applied consistently to test data.

10-Step Preprocessing Pipeline
1

Load & Clean Data

Load TravisTorrent CSV (3.7M+ builds). Verify schema and initial data quality.

2

Handle Null Values

Numeric: forward/backward fill or interpolation. Categorical: mode imputation. Drop rows with >30% nulls.

3

Identify Feature Types

Separate numerical and categorical features. Detect ordinal, nominal, and binary categories.

4

Classify Categorical Features

Categorize by cardinality: Ordinal (order matters), Low-cardinality (<10 unique), High-cardinality (>10 unique).

5

Extract Cyclic Features

Transform timestamp features to sin/cos representation to preserve cyclic nature (hour, day, month).

6

Stratified Train-Test Split

80/20 split preserving class distribution. Fit LabelEncoder on y_train ONLY (no leakage).

7

Encode Categorical Features

Ordinal → OrdinalEncoder. Low-cardinality → OneHotEncoder. High-cardinality → TargetEncoder with smoothing.

8

Encode Target Variable

Binary target (build_successful): True/False → 1/0 using LabelEncoder fitted on training set.

9

Normalize Features

MinMaxScaler: scale all features to [0, 1] range for consistency. Fit on training set, transform test set.

10

Save Artifacts

Export: train_processed.csv, test_processed.csv, all encoders/scalers as .pkl files for inference.

⚠️ Features Dropped (Leakage Prevention)

The following features are deliberately dropped to prevent data leakage:

  • Identifiers: tr_build_id, tr_job_id, build_number, git hashes
  • Post-run Metrics: tr_duration, tr_log_*_tests, test results (only available AFTER build)
  • Status Columns: tr_status, tr_status_* (direct encoding of target)
  • Log Analysis: tr_log_status, tr_log_analyzer_* (requires build completion)
  • Temporal: tr_previous_build info (not available at prediction time)
🔐 Encoding Strategies

1. OrdinalEncoder (Ordinal Features)

For features where order matters (e.g., project maturity: Early → Stable).

Python: from sklearn.preprocessing import OrdinalEncoder
encoder = OrdinalEncoder(handle_unknown='use_encoded_value', unknown_value=-1)
X_train_encoded = encoder.fit_transform(X_train[ordinal_cols])
X_test_encoded = encoder.transform(X_test[ordinal_cols])

2. OneHotEncoder (Low-Cardinality Categorical)

For categorical features with <10 unique values.

Python: from sklearn.preprocessing import OneHotEncoder
encoder = OneHotEncoder(sparse_output=False, handle_unknown='ignore')
X_train_encoded = encoder.fit_transform(X_train[low_card_cols])
X_test_encoded = encoder.transform(X_test[low_card_cols])

3. TargetEncoder (High-Cardinality Categorical)

For features with >10 unique values. Smoothing prevents overfitting to rare categories.

Formula: Smoothed Mean = (Category Mean × Count + Global Mean × α) / (Count + α)
where α (alpha) is the smoothing parameter (default = 1.0)

4. MinMaxScaler (Numerical Features)

Normalize all numerical features to [0, 1] range.

Python: from sklearn.preprocessing import MinMaxScaler
scaler = MinMaxScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)

5. Cyclic Encoding (Temporal Features)

Preserve periodicity of temporal features using sin/cos transformation.

Python: import numpy as np
# For hour of day (0-23)
df['hour_sin'] = np.sin(2 * np.pi * df['hour'] / 24)
df['hour_cos'] = np.cos(2 * np.pi * df['hour'] / 24)

  • ✓ All scalers fitted on training set ONLY
  • ✓ Test set transformed with training parameters
  • ✓ Post-run metrics explicitly dropped
  • ✓ Identifiers and hashes removed before modeling
  • ✓ Stratified split preserves class distribution

📊 Python Inference (Programmatic)

Load Model & Make Predictions

python: import pickle
import numpy as np
import pandas as pd

# Load model and preprocessing artifacts
model = pickle.load(open('src/saved_models/best_model_decision_tree.pkl', 'rb'))
scaler = pickle.load(open('src/saved_models/minmax_scaler.pkl', 'rb'))
target_encoder = pickle.load(open('src/saved_models/target_encoder.pkl', 'rb'))
label_encoder = pickle.load(open('src/saved_models/target_label_encoder.pkl', 'rb'))

# Prepare input
raw_data = {
'git_diff_src_churn': 50,
'gh_test_lines_per_kloc': 200.5,
'gh_team_size': 10
}

# Preprocess
input_df = pd.DataFrame([raw_data])
encoded = target_encoder.transform(input_df)
scaled = scaler.transform(encoded)

# Predict
prediction = model.predict(scaled)[0] # 0 or 1
probability = model.predict_proba(scaled)[0] # [P(fail), P(pass)]

# Interpret
result_text = label_encoder.inverse_transform([prediction])[0]
confidence = max(probability)

print(f"Result: {result_text}")
print(f"Confidence: {confidence:.1%}")
print(f"P(PASS): {probability[1]:.1%}")
print(f"P(FAIL): {probability[0]:.1%}")

🔁 Key Workflow Decisions

Why Stratified Train-Test Split?

Ensures both train and test sets have similar class distributions, preventing skewed evaluations.

Why Fit Encoders on Training Data Only?

Prevents data leakage. Test set is transformed with training parameters to simulate real-world inference.

Why Cyclic Encoding for Temporal Features?

Preserves the periodic nature of time (hour 23 is close to hour 0, not far). sin/cos representation prevents artificial discontinuities.

Why Bayesian Optimization (Optuna)?

More efficient than grid search. Learns from previous trials to intelligently sample hyperparameter space.

Why Decision Tree?

Interpretable, handles non-linearity, performs well on this dataset, faster inference than ensemble methods.

Make a Single Prediction

import pickle, numpy as np, pandas as pd model = pickle.load(open('src/saved_models/best_model_decision_tree.pkl', 'rb')) scaler = pickle.load(open('src/saved_models/minmax_scaler.pkl', 'rb')) encoder = pickle.load(open('src/saved_models/target_encoder.pkl', 'rb')) raw = {'git_diff_src_churn': 50, 'gh_test_lines_per_kloc': 200.5, 'gh_team_size': 10} df = pd.DataFrame([raw]) encoded = encoder.transform(df) scaled = scaler.transform(encoded) pred = model.predict(scaled)[0] # 0=FAIL, 1=PASS proba = model.predict_proba(scaled)[0] print(f"Prediction: {'PASS' if pred == 1 else 'FAIL'}") print(f"Confidence: {max(proba):.1%}")
